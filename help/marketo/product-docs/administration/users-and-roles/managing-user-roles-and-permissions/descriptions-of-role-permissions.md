# Beschreibungen der Rollenberechtigungen {#descriptions-of-role-permissions}

Nachfolgend finden Sie eine Liste aller verfügbaren Berechtigungen, die Sie Ihren Rollen zuweisen können. Berechtigungen sind in der Regel mit bestimmten Funktionsbereichen in Marketo verknüpft und können Ihnen helfen, zu steuern, auf welche Bereiche und Funktionen verschiedene Benutzer zugreifen können.

Weitere Informationen zu Berechtigungen:

* Die Berechtigung &quot;Zugriff&quot;gibt eine Rollenberechtigung für die Ansicht und bearbeitet diesen Teil der Anwendung manchmal.
* Damit eine Rolle Zugriff auf die Unterberechtigungen hat (&quot;Erstellen&quot;, &quot;Löschen&quot;usw.), muss diese Rolle über die Berechtigung &quot;Zugriff&quot;für diesen Teil der Anwendung verfügen. Wenn Sie z. B. jemandem Berechtigungen für die Bearbeitung von Kampagnen erteilen möchten, müssen sie über die Gesamtberechtigung zum Zugriff auf Marketing-Aktivitäten verfügen.
* Möglicherweise können Sie Aktionen oder Assets anzeigen, für die Sie nicht über die entsprechende Berechtigung verfügen. Wenn Sie jedoch versuchen, auf sie zuzugreifen, wird Ihnen eine Warnmeldung über Ihren eingeschränkten Zugriff angezeigt.

## Verfügbare Berechtigungen {#available-permissions}

Wenn Sie [eine Rolle](/help/marketo/product-docs/administration/users-and-roles/managing-user-roles-and-permissions.md) erstellen oder bearbeiten, können Sie die folgenden Berechtigungen auswählen, um diese Rolle zuzulassen, indem Sie die entsprechenden Kontrollkästchen aktivieren.

## Auf Administrator zugreifen  {#access-admin}

Ansicht und nehmen Sie Änderungen an den Einstellungen im Abschnitt &quot;Mein Konto&quot;von Admin vor.

* Zugriffsprüfungsprotokoll - Ermöglicht Benutzern Zugriff auf den Asset-Prüfpfad und den Admin-Prüfpfad
* Kanal aufrufen - Ermöglicht Benutzern nur Zugriff auf die Änderung des Kanal-Tags, nicht auf andere benutzerdefinierte Tags
* Kommunikationsbeschränkung für den Zugriff - Ermöglicht Benutzern Zugriff auf die Aktivierung einer Kommunikationsbeschränkung in Admin
* Access CRM - Ermöglicht Benutzern Zugriff auf CRM, wie Salesforce oder Microsoft Dynamics, in Admin
* Zugriff auf [Data.com](https://Data.com) - Gewährt Benutzern Zugriff auf die Datenfluss-Aktion Data.com
* E-Mail-Admin aufrufen - Ermöglicht Benutzern, E-Mail-Admin zu ändern, um Standardeinstellungen wie Abmelden und Branding-Domänen zu ändern
* Access Ereignis Partners - Ermöglicht Benutzern Zugriff auf LaunchPoint in Admin
* Feldverwaltung aufrufen - Ermöglicht Benutzern Zugriff auf Feldverwaltung in Admin
* Dateiupload - Ermöglicht Benutzern das Hochladen von Bildern und Dateien in das Design Studio
* Landingpages aufrufen - Ermöglicht Benutzern Zugriff auf Landingpages in Admin
* Zugriffsposition - Ermöglicht Benutzern den Zugriff auf Standort in Admin zum Festlegen der Standardsprache, des Gebietsschemas, der Zeitzone und der Währung
* Anmeldeverlauf aufrufen - Gewährt Benutzern Zugriff auf den Verlauf der Benutzeranmeldung im Prüfprotokoll
* Zugriffseinstellungen für die Anmeldung - Ermöglicht Benutzern Zugriff auf die Anmeldeeinstellungen in &quot;Admin für Sicherheit&quot;, &quot;IP-Einschränkungen&quot;und &quot;Bericht für intelligente Liste&quot;
* Zugriff auf benutzerdefinierte Marketo-Aktivität - Ermöglicht Benutzern Zugriff auf benutzerdefinierte Marketo-Aktivitäten in Admin
* Zugriff auf benutzerdefinierte Marketo-Objekte - Ermöglicht Benutzern Zugriff auf benutzerdefinierte Marketo-Objekte in Admin
* Zugriff auf Munchkin - GIs ermöglicht Benutzern den Zugriff auf Munchkin in Admin zum Festlegen von Trackingcode, Personenverfolgung und Aktivieren der API-Konfiguration
* Access Revenue Cycle Analytics - Ermöglicht Benutzern Zugriff auf &quot;Revenue Cycle Analytics&quot;in Admin, um die Synchronisierung der Zusammenfassung und Zuordnung festzulegen
* Zugriffsrollen - Ermöglicht Benutzern Zugriff auf die Verwaltung und Bearbeitung von Rollen, nicht jedoch auf Benutzer
* Zugriff auf Sales Insight - Gewährt Benutzern Zugriff auf die Verwaltung von Sales Insight in Admin, zum Festlegen von Status, API-Konfiguration, Personensortierung und anderen Einstellungen
* Zugriff auf Single-Sign-On - Ermöglicht Benutzern Zugriff auf die Verwaltung von Single-Sign-On in Admin, zum Aktivieren von SAML und zum Arbeiten mit SAML-Einstellungen und Umleitungs-Seiten-URLs
* Zugriff auf intelligente Kampagne - Ermöglicht Benutzern den Zugriff auf intelligente Kampagne in Admin, um Beschränkungen für qualifizierte Mitarbeiter einzuschränken
* Zugriff auf SOAP-API - Ermöglicht Benutzern Zugriff auf die Verwaltung von SOAP-APIs in Web-Services in Admin
* Zugriff auf Tags - Ermöglicht Benutzern Zugriff auf alle benutzerdefinierten Tags mit Ausnahme des Kanal-Tags
* Zugriff auf Schatztruhe - Ermöglicht Benutzern Zugriff auf die experimentellen Funktionen der Schatztruhe in Admin
* Benutzer aufrufen - Ermöglicht Benutzern Zugriff auf die Bearbeitung und Verwaltung von Benutzern (jedoch nicht Rollen) in Admin
* Zugriff auf Webhooks - Ermöglicht Benutzern das Einrichten von Details und Antwortzuordnungen in Admin
* Zugriff auf Arbeitsbereiche und Partitionen - Ermöglicht Benutzern Zugriff auf das Erstellen, Bearbeiten und Löschen von Arbeitsbereichen und Partitionen in Admin
